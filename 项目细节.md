## 3. 语义向量编码器

| 选项 | 实现 | 说明 |
|------|------|------|
| Hash（默认） | `sklearn.HashingVectorizer`, char n-gram(2,4), 2560-d | 确定性哈希，无 GPU 需求 |
| Qwen | `Qwen/Qwen3-Embedding-4B`, 4-bit, Mean Pooling + L2 | 真实语义，需 GPU 加载 |

环境变量：
```powershell
$env:KUAIREC_TEXT_ENCODER = "hash"   # 或 "qwen"
```

Hash 在当前的 DIN + Semantic Attention 架构下效果更好（稀疏投影与 attention 更适配）。

## 5. Student 推理优化

### 推理策略（两阶段 fallback）
1. Whole-JSON：一次生成完整 5 字段 JSON（max_new_tokens=1280）
2. Field-wise（兜底）：逐字段独立生成（max_new_tokens=160 × 5）

### JSON 容错机制
1. `parse_json_content()`：标准 JSON 解析 → raw_decode → 截取 {} 范围
2. `repair_truncated_json()`：控制字符移除 → 转义引号修复 → 末位补全 → 正则字段提取兜底
3. `_fallback_normalize()`：递归搜索 JSON 任意层级提取 5 字段

### 命中率演变
| 版本 | max_tokens | 修复机制 | 命中率 |
|------|:---:|------|:---:|
| V1 | 640 | 无 | 50% |
| V2 | 1280 | 尾部截断修复 | 65% |
| V3 | 1280 | 全量容错+正则兜底 | **93%** |

### Resume 机制
脚本会自动检测输出文件中已完成的 user_id，重启时跳过。无需额外参数。

GPU：NVIDIA GeForce RTX 4060 Laptop GPU

## 7. 召回层

### 四路召回通道

| 通道 | 方法 | 训练？ | 备注 |
|------|------|:---:|------|
| Two-Tower | User/Item Tower（MLP），InfoNCE 训练 | **是** | batch=256 训练，Faiss 索引检索 |
| Item2Item | 共现计数 | 否 | |
| Hot | 全局流行度 | 否 | |
| LLM Semantic | Faiss IndexFlatIP，5→3 向量合并 | 否 | 每个 query 搜 top_k=150 |

### 5→3 向量合并
```
V_profile = mean(long_term_intent, life_stage)          → 用户画像查询
V_intent  = mean(psychological_demand, retrieval_suggestions) → 意图查询
V_explore = interest_growth_points                      → 探索查询
```

### 召回融合
加权 Round-Robin，权重：two_tower:1.0, i2i:1.0, hot:0.8, llm:1.2
最终保留 100 个候选进入精排。

### 两塔局限
当前 batch_size=256 仅提供 255 个 in-batch 负样本，InfoNCE loss 收敛于 5.48（接近随机水平 5.545）。需要更大 batch 或全量负采样才能释放两塔潜力。

## 8. 精排层

### 架构
```
DIN Attention → [Base Features]
                    ↓
              Semantic Cross-Attention (2560→64 projection + MHA + residual gate)
                    ↓
              MMoE (4 experts + 2 gates) → CTR Tower + CVR Tower
```

### 训练配置（V5 最终版）
```text
epochs：15（带 early stop，patience=5）
batch_size：128
lr：1e-3
CTR 正样本阈值：watch_ratio >= 0.8（经 0.3/0.6/0.8 三组对比验证最优）
CTR 负样本阈值：watch_ratio <= 0.3
Loss：BCE（CTR + CVR）
```

---

## 9. CTR/CVR 标签定义

| 标签 | 定义 | 来源 |
|------|------|------|
| CTR positive | watch_ratio >= 0.8（看完） | 每用户每条交互 |
| CTR negative | watch_ratio <= 0.3（浅层浏览） | |
| CVR positive | watch_ratio >= 1.0（重复观看） | |
| CVR negative | 未重复观看 | |
| CTR AUC label | item 是否在 ground_truth 中 | `compute_ctr_cvr_auc()` |
| CVR AUC label | 同上 | |

**注意**：训练标签（BCE）和 AUC 评估标签是两套体系。训练时用 watch_ratio 阈值二值化，评估时用 ground_truth holdout 作为正样本。
