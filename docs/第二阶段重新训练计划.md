## 一、滑动窗口策略升级

### 旧版（Phase1）做法及问题

- Block1 每天都是一个 anchor_date，每用户最多生成 8 个窗口
- 用 `--max_windows_per_user 1` 强制截断，只保留**最早**的 1 个窗口
- **问题**：最早窗口的 history 最少（刚好 ≥10 条），Teacher 看到的用户行为信息量最少，不利于生成高质量用户状态

### 新版策略

**原则**：不是每一天都做 anchor，而是按 block 内部均匀间隔选取锚点，确保每个窗口的 history 足够丰富。

**Block1（8 天：7/5 ~ 7/12）→ 2 个窗口：**

```
窗口 1：anchor=7/7
  history = 7/5 + 7/6（过去所有交互）
  predict = 7/8 当天的交互（跳过 7/7 当天）

窗口 2：anchor=7/11
  history = 7/5 ~ 7/10（过去所有交互）
  predict = 7/12 当天的交互
```

**Block2（10 天：8/1 ~ 8/10）→ 3 个窗口：**

最后一天（8/10）只用于预测，剩余 9 天均匀划分：

```
窗口 1：anchor=8/3
  history = 8/1 + 8/2
  predict = 8/4 当天的交互

窗口 2：anchor=8/6
  history = 8/1 ~ 8/5
  predict = 8/7 当天的交互

窗口 3：anchor=8/9
  history = 8/1 ~ 8/8
  predict = 8/10 当天的交互
```

**Block3（10 天：8/27 ~ 9/5）→ 3 个窗口：**

同样逻辑（评估集，仅用于最终测试）：

```
窗口 1：anchor=8/29, predict 8/30
窗口 2：anchor=9/1,  predict 9/2
窗口 3：anchor=9/3,  predict 9/5（注：跳过 9/4 的 anchor，history=8/27~9/3）
```

**每用户样本数**：训练集 = Block1(2) + Block2(3) = **5 条**；测试集 = Block3(3) = **3 条**

## 二、训练/验证/测试集划分方案

### 关键发现

**Small_matrix 的 1411 用户 100% 包含在 big_matrix 的 7176 用户中。** 这意味着 small_matrix 可以直接用作验证集。

### 推荐方案：双实验对照设计

核心思路：**通过控制 ID embedding 的可用性，量化 LLM 语义特征在 warm-start 和 cold-start 两种场景下的独立贡献。**

---

#### 实验 1：Warm-Start（ID 可用）

```
训练集：全部 7176 用户 × block1(2)+block2(3) = 5 窗口/用户
测试集：全部 7176 用户 × block3(3) = 3 窗口/用户

DIN-only：ID embedding + 行为特征
LLM-full：ID embedding + 行为特征 + LLM 语义特征

回答的问题：在"模型认识用户"的前提下，LLM 能额外提升多少？
```

#### 实验 2：Cold-Start（ID 不可用）

```
训练集：排除 small 的 ~5765 用户 × block1(2)+block2(3) = 5 窗口/用户
测试集：small 的 1411 用户 × small_matrix（全观测）

DIN-only（冷启动版）：无用户 ID embedding，仅靠 item embedding + 全局统计
LLM-full（冷启动版）：无用户 ID embedding，靠 LLM 语义特征 + item embedding

回答的问题：当模型完全没见过一个用户时，LLM 语义特征能兜底多少？
          —— 这正是真实冷启动场景（新用户注册后没有任何行为）。
```

#### 实验 2 的关键设计

**冷启动用户（small 的 1411 人）从未出现在训练集中，ID embedding 不可用。处理方式：**

1. **User ID Embedding**：冷启动用户全部映射到 `<UNK>` token，其 embedding 用所有训练用户 ID embedding 的**均值**初始化（而非随机）。这相当于"在没有个性化信息之前，先假设他是一般用户"。

2. **双塔 User Tower**：冷启动用户的 user state 同样取训练用户 user state 的**全局均值**：
   ```
   cold_user_emb = user_tower(train_user_states.mean(axis=0))
   ```
   这保证新用户在没有历史时也能被召回通道正常检索。

3. **LLM 语义特征**：不受影响——Student 从用户的少量行为生成状态描述，不依赖用户 ID。这是冷启动场景下 LLM 特征的独立价值所在。

**设计合理性**：用全局均值做先验是工业界冷启动的标准做法——"在没有个性化信息之前，假设你是个普通人"。如果 LLM 语义特征能在这个基线上显著提升效果，说明语义理解对冷启动有独立贡献。

#### 实验 1 + 实验 2 的对照价值

```
Warm-start 的 LLM ∆  = 实验1的 LLM-full - DIN-only → ID+LLM vs 纯ID
Cold-start 的 LLM ∆  = 实验2的 LLM-full - DIN-only → LLM vs 全局统计

两个 ∆ 的差值 = LLM 语义特征在冷启动场景下的"独立贡献"
如果 Cold-start ∆ > Warm-start ∆，说明 LLM 在冷启动时价值更大
（预期如此，因为冷启动时 ID 信息缺失，语义特征的边际价值更高）
```

#### 附：Small_matrix 全观测验证

实验 2 天然用了 small_matrix 的全观测特性——每个 user-item 对都有 watch_ratio，不存在"没曝光=不喜欢"的偏差。这回答了推荐系统评估中最核心的可信度问题：**模型是真的推荐准了，还是因为训练数据本身就偏了？**

## 五、Student Prompt 升级方案

Phase1 的 compact schema（5 字段各 1-2 句）是因为 487 条训练数据太少，必须简化格式保证稳定性。

数据量扩到 10,000+ 条后，建议升级为 **compact-rich schema**——保持平铺 JSON 结构（稳定），但每字段信息量升级：

| 字段 | Phase1（compact） | 第二阶段（compact-rich） |
|------|------|------|
| long_term_intent | 1-2 句 | **3-4 句**：具体内容簇 + 消费深度 + 近远期变化趋势 |
| life_stage | 1 句，保守判断 | **2-3 句**：推测依据 + 置信度 + 关键支撑行为 |
| psychological_demand | 1-2 句 | **2-3 句**：核心需求 + 近期动机 + 与长期需求的差异 |
| retrieval_suggestions | 1-2 句 | **3-4 句**：分显式查询 / 隐式召回 / 探索方向三项 |
| interest_growth_points | 1-2 句 | **3-4 句**：新兴信号 + 桥接概念 + 具体探索主题推荐 |

理由：平铺 JSON 保证格式稳定性（Phase1 已验证 93% 命中率），字段内容扩充利用 10,000 条数据的训练量。不冒进到嵌套 JSON（2B 模型结构性天花板）。
