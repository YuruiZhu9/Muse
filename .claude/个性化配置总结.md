## 🔧 Skills & MCP 工具集成

### 何时推荐创建 Skill

当以下情况发生时，我会主动问你：

1. **复用性识别** - 同一类请求出现 3 次以上
2. **模式识别** - 你有特定的使用习惯
3. **工作流固化** - 某个流程稳定可复用
4. **专业知识** - 涉及特定领域知识

### 推荐的 Skills

| Skill | 用途 |
|-------|------|
| `/commit` | 代码提交 |
| `/commit-push-pr` | 提交并创建 PR |
| `frontend-design` | 前端设计 |

## 📋 四阶段工作流

### Phase 1: 研究 (Research)
```bash
/research_codebase "功能描述" "研究范围"
```
- 理解代码库现状
- 4 个并行研究方向
- 生成研究文档

### Phase 2: 计划 (Plan)
```bash
/create_plan "功能名称"
```
- 迭代 5-6 次（黄金分割点）
- 分阶段规划
- 明确验证标准

### Phase 3: 实现 (Implement)
```bash
/implement_plan phase:1
```
- 一次只做一阶段
- 完成验证后再下一阶段
- 进度保存在文件中

### Phase 4: 验证 (Validate)
```bash
/validate_plan "功能名称"
```
- 对比计划 vs 实际
- 生成验证报告
- 安全网检查
